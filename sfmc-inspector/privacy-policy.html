SFMC Inspector Reloaded

Privacy Policy

Last updated: April 2026
Summary: SFMC Inspector Reloaded does not collect, store, or transmit any personal data to external servers. All data stays within your browser and your Salesforce Marketing Cloud account.

What SFMC Inspector Reloaded does

SFMC Inspector Reloaded is a browser extension that helps Salesforce Marketing Cloud (SFMC) developers and administrators explore Data Extensions, Automations, Journeys, and SQL relationships — all from within the SFMC interface.

Data we access

When you use SFMC Inspector Reloaded on a Salesforce Marketing Cloud page, the extension:

  • Reads your existing browser session to authenticate with SFMC on your behalf
  • Makes API calls to your SFMC account to retrieve metadata (Data Extension names, Automation details, Journey structures, Query Activity SQL)
  • Indexes and searches Query Activity metadata locally in your browser

The extension only accesses data that your SFMC user account has permission to see. It cannot access more than what you can access through the standard SFMC interface.

Data we do NOT collect

  • No personal data is collected or stored outside your browser
  • No data is sent to any server other than your own Salesforce Marketing Cloud instance
  • No analytics, tracking, or telemetry of any kind
  • No cookies are created beyond what SFMC itself uses
  • No user accounts, registration, or login is required

Local storage

SFMC Inspector Reloaded uses Chrome's session storage (chrome.storage.session) to temporarily cache your SFMC session context (hostname, stack key, and authentication token) for the duration of your browser session. This data is:

  • Stored locally in your browser only
  • Cleared automatically when you close Chrome
  • Never transmitted to any external server

Third-party services

SFMC Inspector Reloaded does not use any third-party services, analytics platforms, or external APIs. The only network requests made are directly to your Salesforce Marketing Cloud account's own API endpoints (*.exacttarget.com, *.marketingcloudapis.com).

Permissions

The extension requests the following Chrome permissions:

  • scripting / activeTab — to execute authenticated API calls directly within the active SFMC tab; the browser's existing session is used automatically, without reading or storing cookies
  • storage — to temporarily cache session context locally in the browser
  • tabs — to find open SFMC tabs for session detection
  • host permissions for *.exacttarget.com, *.marketingcloudapis.com, *.marketingcloud.com and *.marketingcloudapps.com — to communicate with SFMC APIs

No permission is used beyond its stated purpose.

Open source

SFMC Inspector Reloaded is open source. You can inspect the full source code to verify these claims independently.
